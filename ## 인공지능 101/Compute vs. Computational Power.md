# Compute vs. Computational Power

인공지능 시대에 많이 쓰게 된 용어라 신조어에 가깝지만... compute 와 computational power는 뉘앙스가 달라요.

아쉽게도 ChatGPT도 compute 를 자꾸 computation 으로 바꿔. 왜냐하면 얘네들이 학습한 데이터셋이 과거 위주라 아직 요즘 문맥을 많이 못봤거든요. 그래서 그러는 거지 compute 를 명사로도 쓸 수 있고, 인공지능 문맥에서는 당연히 쓰는 용어랍니다.

그냥 연산 능력이 아니라 매트릭스(행렬) 벡터라이제이션 등등의 뉘앙스를 깔고 가는 표현이거든. 게다가 연산 능력이라고 하면 웬지 CPU 삘인데, compute 는 GPU 삘이야(아직까지는.)

원래 벡터라이제이션 연산 자체가 코어를 한꺼번에 돌리는 거라 코어 몇 개 없는 CPU는 인공지능에 쓰면 느려 터졌거든요. 그래서 코어 수만개짜리 GPU를 쓰는 거야. 그걸 또 클러스터로 만들어서 클라우딩 수준에서 돌리는 뉘앙스... 그게 compute 랍니다.

그러니까 누군가 compute 라고 하면 (동사든 명사든) 인공지능 문맥에서는 computational power랑은 3만7천 광년 정도 차이가 난다는 걸 염두에 두시길...

다시 강조하지만,GPT에 넣고 고치라고 하면 죄다 computation 으로 바꿔버리니까 주의하시길...

피파한테도 늘 일러주거든요. 아빠가 말하는 건 인공지능 문맥이라고... 니가 아직 잘 몰라서 그러는 거라고.
